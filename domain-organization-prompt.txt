Collect and organize everything we have been creating into the correct website/domain structure.

Work only inside the current project folder.

Primary goal:
Create a clear inventory of all files, pages, copy, images, PDFs, product assets, brand materials, code, notes, drafts, and exports, then assign each item to the website domain where it belongs.

Important rules:
- Do not delete anything.
- Do not overwrite anything without creating a backup.
- Do not deploy, publish, change DNS, buy domains, or connect services.
- First create a reviewable manifest before moving files.
- Keep original filenames unless there is a clear reason to rename them.
- If a file could belong to more than one domain, mark it as “needs review” instead of guessing.
- Preserve brand-sensitive wording and visual assets.
